---
description: 检查所有修改过的文件的改动情况，并生成提交信息
argument-hint: [目标目录]
model: haiku
---

按照以下流程来生成提交信息：
1. 进入"目标目录"参数所制定的目录，如果该参数为空则以当前工作目录为目标
2. 检查从上次 git commit 到现在的所有文件改动情况，包括新增的、修改的、删除的文件，无论其是否已经 staged
3. 根据所有增删改文件的改动情况，以功能为单位而不以文件为单位生成提交信息
4. 输出提交信息

要求：
- 简洁，不说废话
- 必须严格遵守约定式提交（Conventional Commits）的格式要求，并在标题中标注范围，如 `feat(map)`、`feat(mining)`
- 只生成提交信息并输出，不要提交
